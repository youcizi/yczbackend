# 🧩 系统插件开发指南 (V2.0 - 自动发现架构)

本系统采用 **“全动态、自愈式”** 的插件架构。开发者只需在 `src/plugins/` 线下建立符合规范的目录，系统将在构建/开发阶段自动扫描并接入路由、数据库 Schema 及管理端 UI。

---

## 一、 插件物理结构 (Directory Contract)

插件必须作为一个完整的物理目录存在于 `src/plugins/` 路径下。

### 标准目录结构
```text
src/plugins/[your-plugin-slug]/
├── manifest.ts       # 【必须】插件身份元数据与权限声明
├── index.ts          # 【必须】后端逻辑出口（导出 Hono 子应用）
├── admin/            # 管理后台资产
│   └── index.tsx     # 【必须】管理端 React UI 入口
├── schema/           # 数据模型定义
│   └── [table].ts    # 【自动扫描】Drizzle Schema 定义文件
└── services/         # 插件私有业务逻辑
```

---

## 二、 核心组件规范

### 1. 身份清单 (`manifest.ts`)
定义插件的基本信息及 RBAC 权限条目。系统会自动将 `permissions` 同步至全域权限库。

```ts
export const MANIFEST = {
  slug: 'membership',     // 必须与文件夹名一致
  name: '核心会员系统',
  description: '提供多语种等级管理、折扣权益及 D1 架构支持',
  version: '2.1.0',
  author: 'YourTeam',
  // 插件专属权限（系统自动注册）
  permissions: [
    { slug: 'membership.manage', name: '管理会员等级', perm_category: '会员系统' },
    { slug: 'membership.view', name: '查看等级数据', perm_category: '会员系统' }
  ],
  // 管理后台菜单挂载
  adminMenu: {
    icon: 'Users',
    path: '/admin/plugins/membership'
  }
};
```

### 2. 逻辑出口 (`index.ts`)
导出两个 Hono 子应用。系统会自动将它们分别路由至 **管理端代理** 和 **商城端枢纽**。

```ts
import { Hono } from 'hono';
import { membershipAdminApp } from './api-admin';
import { membershipStorefrontApp } from './api-front';

export default {
  /** [ADMIN] 挂载至 /api/v1/plugins/proxy/[slug]/* */
  admin: membershipAdminApp,
  
  /** [CLIENT] 挂载至 /api/v1/s/[slug]/* */
  storefront: membershipStorefrontApp,
  
  /** [INIT] 插件激活时的初始化钩子 */
  init: () => { console.log('Membership Plugin Activated'); }
};
```

### 3. 数据模型 (`schema/`)
所有在该目录下的 `.ts` 文件都会被扫描器捕捉并自动通过生成代码导出至 `src/db/schema.ts`。

## 三、 集成与部署流水线

### 1. 自动发现机制
系统运行 `npm run dev` 或 `npm run build` 时，会首执行 `scripts/sync-plugins.mjs`：
- **Schema 同步**：生成 `src/db/auto-schema.gen.ts`。
- **路由注册**：生成 `src/lib/auto-registry.gen.ts`。
- **主系统解耦**：`app.ts` 和 `schema.ts` 无需手动 import 任何插件代码。

### 2. 激活流程
1. 将插件目录放入 `src/plugins/`。
2. 重启开发环境（触发扫描）。
3. 进入 **“插件管理中心”** $\rightarrow$ **“手动登记新资产”**。
4. 输入对应的 `Slug`（文件夹名）。
5. 开启 **“运行”** 开关。

---

## 四、 关键网关说明

- **管理端网关 (`/api/v1/plugins/proxy/:slug/*`)**:
  - 自动注入 `admin_session`。
  - 需要 `plugins.manage` 权限。
  - 由 `plugins.ts` 动态转发。

- **商城端网关 (`/api/v1/s/:slug/*`)**:
  - 自动解析 `current_member` 上下文。
  - 由 `storefront-hub.ts` (Storefront Hub) 统一分发。
  - 具备状态预检：若插件在后台关闭，网关自动返回 404。

---

## 五、 环境变量与初始化

在“手动登记资产”弹窗中提供的 **“高级初始化配置 (JSON)”**：
- 它作为插件的“环境变量”存储在 D1 数据库中。
- 开发者可以在插件代码中通过 `PluginService.getPluginConfig(slug)` 动态读取。
- 典型用途：存储第三方 API Key、配置非标准资源路径等。

---

> **⚠️ 注意事项**: 
> 1. 禁止在插件代码中直接引入主系统的 `import { db } from '../../db'` 以免造成循环依赖。
> 2. 请使用动态传入的 D1 绑定或通过 Service 模式获取数据库实例。
> 3. 修改插件 UI 后，由于使用了 `lazy()` 加载，可能需要刷新浏览器查看变更。
